# Руководство по редактированию контента

## Обзор

Проект использует **прямое редактирование JSON файлов** без CMS. Все данные хранятся в папке `src/content/` в виде JSON файлов.

## Структура контента

### 📁 Папка `src/content/`

Все файлы контента находятся в папке `src/content/`:

- `hero.json` - слайды главной страницы
- `performances.json` - спектакли
- `publications.json` - публикации и новости
- `documents.json` - документы
- `media.json` - медиафайлы
- `contacts.json` - контактная информация
- `about.json` - страница "О театре"
- `privacy-policy.json` - политика конфиденциальности
- `seo.json` - SEO метаданные для всех страниц
- `faq.json` - часто задаваемые вопросы
- `site.json` - глобальные настройки сайта

## Как редактировать контент

### 1. Откройте нужный JSON файл

Например, для редактирования спектаклей откройте:
```
src/content/performances.json
```

### 2. Редактируйте данные

JSON файлы имеют понятную структуру. Например, для спектакля:

```json
{
  "id": 1,
  "title": "Название спектакля",
  "duration": "2 часа",
  "ageGroup": "16+",
  "description": "Описание спектакля...",
  "cast": "Актерский состав",
  "creators": {
    "director": "Режиссер",
    "writer": "Сценарист"
  },
  "mainImage": "/images/performances/spectacle1.jpg",
  "gallery": [
    "/images/performances/gallery1.jpg",
    "/images/performances/gallery2.jpg"
  ]
}
```

### 3. Сохраните файл

После редактирования сохраните файл (Ctrl+S).

### 4. Проверьте изменения локально

```bash
npm run dev
```

Откройте http://localhost:5175/ и проверьте изменения.

### 5. Опубликуйте изменения

```bash
git add .
git commit -m "Update content"
npm run deploy
```

## Детальное описание файлов

### 🎭 `performances.json` - Спектакли

```json
{
  "performances": [
    {
      "id": 1,
      "title": "Название спектакля",
      "duration": "2 часа",
      "ageGroup": "16+",
      "description": "Описание спектакля",
      "cast": "Актерский состав",
      "creators": {
        "director": "Режиссер",
        "writer": "Сценарист"
      },
      "mainImage": "/images/performances/spectacle1.jpg",
      "gallery": [
        "/images/performances/gallery1.jpg",
        "/images/performances/gallery2.jpg"
      ]
    }
  ]
}
```

**Поля:**
- `id` - уникальный номер спектакля
- `title` - название спектакля
- `duration` - продолжительность
- `ageGroup` - возрастная категория
- `description` - описание спектакля
- `cast` - актерский состав
- `creators.director` - режиссер
- `creators.writer` - сценарист
- `mainImage` - основное изображение
- `gallery` - галерея изображений

### 📰 `publications.json` - Публикации

```json
{
  "publications": [
    {
      "id": 1,
      "title": "Заголовок публикации",
      "date": "2025-01-15",
      "image": "/images/publications/news1.jpg",
      "tags": ["новости", "театр"],
      "html": "<p>Содержимое публикации в HTML формате</p>",
      "gallery": [
        "/images/publications/gallery1.jpg"
      ]
    }
  ]
}
```

**Поля:**
- `id` - уникальный номер публикации
- `title` - заголовок
- `date` - дата в формате YYYY-MM-DD
- `image` - изображение публикации
- `tags` - теги (массив строк)
- `html` - содержимое в HTML формате
- `gallery` - галерея изображений (опционально)

### 🏠 `hero.json` - Слайды главной страницы

```json
{
  "slides": [
    {
      "title": "Заголовок слайда",
      "description": "Описание слайда",
      "backgroundImage": "/images/hero/slide1.jpg",
      "buttons": [
        {
          "text": "Текст кнопки",
          "url": "/performances",

```

**Поля:**
- `title` - заголовок слайда
- `description` - описание
- `backgroundImage` - фоновое изображение
- `buttons` - массив кнопок
  - `text` - текст кнопки
  - `url` - ссылка
  - `primary` - основной стиль (true/false)

## Работа с изображениями

### Загрузка изображений

1. **Скопируйте изображение** в папку `public/images/`
2. **Укажите путь** в JSON файле: `/images/folder/image.jpg`

### Структура папок для изображений

```
public/images/
├── hero/           # Слайды главной страницы
├── performances/   # Изображения спектаклей
├── publications/   # Изображения публикаций
├── about/          # Изображения для страницы "О нас"
└── logo.png        # Логотип
```

### Рекомендации по изображениям

- **Формат**: JPG для фотографий, PNG для логотипов
- **Размер**: до 2MB для быстрой загрузки
- **Разрешение**: 1200x800px для основных изображений
- **Названия**: используйте понятные имена файлов

## Работа с HTML контентом

### Для публикаций и страницы "О театре"

Используйте HTML теги для форматирования:

```html
<p>Обычный абзац</p>
<h2>Заголовок второго уровня</h2>
<h3>Заголовок третьего уровня</h3>
<strong>Жирный текст</strong>
<em>Курсив</em>
<ul>
  <li>Элемент списка 1</li>
  <li>Элемент списка 2</li>
</ul>
<ol>
  <li>Нумерованный элемент 1</li>
  <li>Нумерованный элемент 2</li>
</ol>
<a href="/link">Ссылка</a>
```

## 🎨 Минималистичные стили для публикаций

Для написания статей используйте только базовые HTML элементы. Все декоративные блоки удалены для минимализма.

### Рекомендуемые элементы

| Элемент | Код | Описание |
|---------|-----|----------|
| **Заголовки** | `<h2>`, `<h3>` | Структура статьи |
| **Параграфы** | `<p>` | Основной текст |
| **Выделение** | `<strong>`, `<em>` | Важная информация |
| **Списки** | `<ul>`, `<ol>`, `<li>` | Перечисления |
| **Ссылки** | `<a href="">` | Навигация |
| **Кнопки** | `<a class="btn-primary">` | Призывы к действию |

### Примеры использования

**Заголовки:**
```html
<h2>Основной раздел</h2>
<h3>Подраздел</h3>
```

**Выделение важной информации:**
```html
<p><strong>Важная информация</strong> в тексте.</p>
<p><em>Курсив для акцента</em>.</p>
```

**Списки:**
```html
<ul>
  <li>Первый пункт</li>
  <li>Второй пункт</li>
</ul>
```

**Призыв к действию:**
```html
<div class="cta-section">
  <a href="/contacts" class="btn-primary">Связаться с нами</a>
</div>
```

## Советы и рекомендации

### ✅ Хорошие практики

1. **Делайте резервные копии** перед большими изменениями
2. **Тестируйте локально** перед публикацией
3. **Используйте понятные названия** для изображений
4. **Проверяйте ссылки** на изображения
5. **Сохраняйте JSON структуру** - не удаляйте обязательные поля

### ❌ Чего избегать

1. **Не удаляйте обязательные поля** из JSON
2. **Не используйте слишком большие изображения**
3. **Не забывайте про кавычки** в JSON
4. **Не нарушайте структуру** массивов и объектов

### 🔧 Решение проблем

#### Ошибка в JSON

Если сайт не загружается, проверьте:
1. Правильность JSON синтаксиса
2. Наличие всех кавычек
3. Правильность запятых

#### Изображения не загружаются

Проверьте:
1. Путь к изображению в JSON
2. Существование файла в папке `public/images/`
3. Правильность расширения файла

#### Изменения не отображаются

1. Убедитесь, что сохранили файл
2. Перезапустите локальный сервер
3. Очистите кэш браузера
