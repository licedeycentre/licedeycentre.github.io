# Руководство по настройке кастомных ссылок для билетов

## Обзор

Теперь кнопка "Приобрести билеты" на спектаклях поддерживает кастомные ссылки. Это позволяет:
- Направлять пользователей на внешние сайты бронирования билетов
- Использовать формы-заявки (например, Яндекс.Формы)
- Ссылаться на социальные сети для заказа билетов
- Использовать любые другие внешние сервисы

## Как это работает

### Логика отображения кнопки

Кнопка "Приобрести билеты" отображается при выполнении всех условий:
1. `showTicketsButton={true}` (включен параметр)
2. `status` равен `'current'` или `'planned'`
3. Есть будущие даты показов (`futureDates.length > 0`)

### Поведение кнопки

- **Если указан `ticketsUrl`**: кнопка ведет на внешнюю ссылку (открывается в новой вкладке)
- **Если `ticketsUrl` не указан**: кнопка ведет на страницу контактов (`/contacts`)

## Настройка в файле performances.json

### Добавление кастомной ссылки

Добавьте поле `ticketsUrl` к любому спектаклю:

```json
{
  "id": 1,
  "title": "Пустоцвет",
  "duration": "45 минут",
  "ageGroup": "12+",
  "description": "...",
  "cast": "...",
  "creators": "...",
  "image": "...",
  "gallery": [...],
  "status": "current",
  "showDates": [...],
  "ticketsUrl": "https://forms.yandex.ru/cloud/your-form-id"
}
```

### Примеры использования

#### 1. Форма-заявка (Яндекс.Формы)
```json
"ticketsUrl": "https://forms.yandex.ru/cloud/68e8929a02848f207c913644"
```

#### 2. Внешний сайт бронирования
```json
"ticketsUrl": "https://example-booking-site.com/performance-name"
```

#### 3. Социальные сети
```json
"ticketsUrl": "https://telegram.me/licedeycentre"
"ticketsUrl": "https://vk.com/licedeycentre"
```

#### 4. WhatsApp
```json
"ticketsUrl": "https://wa.me/79147126952?text=Хочу%20забронировать%20билеты%20на%20спектакль"
```

#### 5. Email
```json
"ticketsUrl": "mailto:nko.licedey@list.ru?subject=Бронирование%20билетов"
```

## Где отображается кнопка

Кнопка "Приобрести билеты" с кастомными ссылками отображается в:

1. **Сетке спектаклей** (`/performances`) - в карточках спектаклей
2. **Странице отдельного спектакля** (`/performance/:id`) - в детальной информации

## Технические детали

### Типы данных

```typescript
export interface Performance {
  // ... другие поля
  ticketsUrl?: string  // Кастомная ссылка для покупки билетов
}
```

### Компоненты

- `PerformanceCard.tsx` - карточка спектакля в сетке
- `PerformancePage.tsx` - страница отдельного спектакля

### Безопасность

Для внешних ссылок автоматически добавляются атрибуты:
- `target="_blank"` - открытие в новой вкладке
- `rel="noopener noreferrer"` - защита от атак

## Рекомендации

1. **Тестируйте ссылки** - убедитесь, что все ссылки работают корректно
2. **Используйте HTTPS** - для безопасности пользователей
3. **Короткие ссылки** - для лучшего UX используйте короткие URL
4. **Мобильная версия** - проверьте работу на мобильных устройствах
5. **Резервный вариант** - если внешний сервис недоступен, пользователи всегда могут связаться через контакты

## Примеры конфигурации

### Спектакль с формой бронирования
```json
{
  "id": 1,
  "title": "Пустоцвет",
  "status": "current",
  "showDates": [
    {"date": "2024-12-15", "time": "19:00"},
    {"date": "2024-12-22", "time": "19:00"}
  ],
  "ticketsUrl": "https://forms.yandex.ru/cloud/example-tickets-form"
}
```

### Спектакль без кастомной ссылки (по умолчанию)
```json
{
  "id": 2,
  "title": "Старые друзья",
  "status": "planned",
  "plannedThisSeason": true
  // ticketsUrl не указан - будет ссылка на /contacts
}
```

### Спектакль с внешним сайтом бронирования
```json
{
  "id": 3,
  "title": "Сказка о потерянном времени",
  "status": "current",
  "showDates": [...],
  "ticketsUrl": "https://example-booking-site.com/skazka"
}
```
